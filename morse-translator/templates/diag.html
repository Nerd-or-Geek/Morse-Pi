⚠ GPIO NOT AVAILABLE

The server cannot access GPIO hardware. This usually means one of the following:

1. Missing backend library
Run on your Pi: sudo apt install python3-lgpio
or sudo apt install python3-rpi.gpio

2. Running on a non-Pi machine
GPIO is only available on Raspberry Pi hardware.

3. Permissions
Try running the server with sudo python3 app.py or add your user to the gpio group:
sudo usermod -a -G gpio $USER

HOW TO WIRE
Paddle / Straight Key (pull-up wiring):
 • Connect the common / ground wire of your paddle/key to any GND pin on the header (e.g. Pi Pin 6, 9, 14, 20, 25, 30, 34, or 39).
 • Connect each signal wire to its assigned BCM GPIO pin.
 • When a paddle is pressed, the signal pin is pulled LOW (grounded) and will glow green above.

Expected behaviour:
 • Key/paddle at rest → pin shows IDLE (HIGH)
 • Key/paddle pressed → pin shows GROUNDED (LOW) ●

If pins stay IDLE even when pressed:
 • Check your GND wire is on a GND pin (not a 3.3V or 5V pin).
 • Check your signal wire is on the correct BCM pin number.
 • Try sudo apt install python3-lgpio and restart the server.
 • Click REINIT above to force GPIO re-initialization.
